# Roadmap

GitHub Milestones and the Project Board are the authoritative planning tools.

This document explains the intent of the current planning structure. It does not replace existing GitHub Issues, Milestones, or board status.

---

# Planning Principles

- Finish the foundation before feature expansion.
- Build vertical slices before scale.
- Validate the first exploration loop before adding more content.
- Keep the MVP small enough to finish.
- Move future ideas to `docs/project/IDEA_VAULT.md`.

---

# Current Planning Sequence

The current milestone intent is:

1. Foundation.
2. Core Gameplay.
3. Vertical Slice.
4. MVP.
5. Launch.

If GitHub Milestone names or assignments differ, GitHub remains authoritative and the difference should be reported.

---

# Foundation

Purpose:

- establish repository structure;
- confirm local setup;
- configure validation;
- document architecture and workflow;
- prepare the project for implementation.

---

# Core Gameplay

Purpose:

- player movement;
- camera behavior;
- interaction system;
- first cache;
- first exploration loop;
- basic game state.

---

# Vertical Slice

Purpose:

- combine movement, interaction, discovery, UI, audio, and visual wonder into one coherent playable slice.

The vertical slice should prove the game can feel magical before adding scale.

---

# MVP

Purpose:

- stabilize the first playable release;
- validate performance;
- polish UX;
- prepare for public or semi-public playtesting.

---

# Launch

Purpose:

- complete QA;
- publish release notes;
- record known issues;
- make a deliberate launch decision.

---

# Planning Review Needed

Existing GitHub Issues were created before this documentation consolidation.

They should be reviewed after Phase 2 to confirm:

- milestone assignment still matches the intended sequence;
- issue scope is not larger than the MVP;
- issue acceptance criteria are specific enough for Codex implementation;
- no required issue is missing for the documented MVP.
